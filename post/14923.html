GitHub + Hexo

发表于 2020-01-20 更新于 2020-05-25 分类于 博客 热度： ℃ 热度： 2k 2 分钟

前言

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown （或其他渲染引擎）解析文章。 Hexo 生成的属于静态网站，体积小，网站的性能更优，而且纯静态的方便网站的 seo。但缺点也是有的，其一，博客部署在 GitHub 上，由于众所周知的原因，国外的服务器在国内访问速度确实不行，解决办法也是有的，部署双节点，即 GitHub 和 coding。其二，因为Hexo 是基于 Nodo.js 的，所以生成文章需要搭建环境才能运行。解决办法即为利用 Daocloud 打造全自动发布的 Hexo 博客。

Node.js 为大多数平台提供了官方的安装程序。

注册 GitHub

  1. 注册 GitHub ，新建项目，在 Repository name 务必要写跟你 GitHub 名称一致，即与前面 owner 中保持一致，并在后面加上 .github.io 。例如我的 GitHub 名称为 josephsun-6 ，此处应该写的为 josephsun-6.github.io 。
  2. 建好后复制 Quick setup 中的地址。
  3. 点击 settings ，向下拉有个 GitHub Pages，点 Choose a theme 选择主题，重新加载页面后， GitHub Pages 就有了博客的页面。

安装 Hexo

  1. 首先安装 Hexo 到本地，在合适的地方新建文件夹，用来存放博客文件。

  2. 右键打开 Git Bash Here ,打开 git 控制台，输入

  3. 安装 Hexo 。有橙色报错没关系，只要不是红色就行。最后输入 hexo -v 验证安装是否成功。

  4. 接下来需要初始化，输入 hexo init 初始化文件夹，接下来 npm install 安装其他组件。

  5. 输入 hexo g 生成静态网页，输入 hexo s 打开本地服务器，然后浏览器打开 http://localhost:4000/，就可以看到博客。

  6. 按 ctrl + c 关闭本地服务器。

绑定域名

  • 默认域名 xxx.github.io ，如果想要专属域名可以去xx云购买，一般也都带解析，打开 GitHub 的博客项目里 settings，在 GitHub Pages 里 Custom domain 填写所购买的域名，此时项目中已有一个 CNAME 的文件。由于每一次上传博客可能会清除此文件，因此可以在本地博客下的 ~\博客\source 目录下，创建 CNAME 文件，里面填写域名。
